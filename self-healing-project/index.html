Self-Healing System

Automatic recovery when the application container goes down.

1

Monitor Container

The heal.sh script checks the Docker container status every 10 seconds using a simple loop.

2

Detect Failure

If the container named "mywebsite" is not running, the script immediately detects the failure.

3

Auto-Restart

The script attempts to restart the container. If it doesn't exist, it creates a brand new one from the latest Docker image.

4

Log Everything

Every action is logged with a timestamp so you can trace exactly what happened and when.

heal.sh
$ bash heal.sh 2026-07-10 10:30:00 [INFO] Checking container... 2026-07-10 10:30:00 [INFO] Container is healthy. 2026-07-10 10:30:10 [INFO] Checking container... 2026-07-10 10:30:10 [INFO] Container is healthy. 2026-07-10 10:30:20 [INFO] Checking container... 2026-07-10 10:30:20 [WARNING] Container stopped. 2026-07-10 10:30:20 [WARNING] Attempting restart... 2026-07-10 10:30:21 [SUCCESS] Container recovered. 2026-07-10 10:30:31 [INFO] Checking container... 2026-07-10 10:30:31 [INFO] Container is healthy.
